### What it fixes
`compressToBytes` (compact-export path) opened the bounds-read stream as:

```kotlin
resolver.openInputStream(uri)?.use {
    BitmapFactory.decodeStream(it, null, boundsOpts)
} ?: return null
```

`?.use { block }` evaluates to the block's result. With `inJustDecodeBounds = true`,
`decodeStream` **always returns null by design** (reads dimensions only, no allocation), so the whole
expression was null and `?: return null` fired on **every** photo — regardless of whether the URI was
readable. Result: compact backups shipped with `photoUris: []` and no `photos/` zip entries, and
`ExportResult.photosSkipped` counted every photo. Now the guard checks the stream itself:

```kotlin
val streamForBounds = resolver.openInputStream(uri) ?: return null
streamForBounds.use {
    BitmapFactory.decodeStream(it, null, boundsOpts)
}
```
