10th map-camera attempt. Brief `docs/code-brief-map-camera-v10.md` verified against live source first

(commit `b7730f7`), brief moved to `docs/DONE/`. `assembleDebug` BUILD SUCCESSFUL (`compileDebugKotlin`
compiled the change).

Changes:
- **Multi-pin zoom:** replaced v9's `newLatLngBounds(bounds, w, h, padding)` (under-zoomed on-device,
  A53 vc43, for our 209° span) with direct **Mercator zoom math** — `lngZoom` from linear longitude px,
  `latZoom` from log-compressed Mercator latitude px (32dp inset each side), take the smaller, coerce
  1f–18f → `newLatLngZoom(center, zoom)`. Antimeridian-correct center (largest-gap, v8 logic) unchanged.
- **Diagnostics:** `Log.d("MapCamera", "v10: …")` of span/zoom/center/map-px; `move()` failures now go
  to `Log.e` instead of being swallowed by `runCatching`. Logging stays in until the map is confirmed
  stable (debug-only, no user-visible change).
- **Imports:** removed `LatLngBounds`; added `android.util.Log`, `kotlin.math.ln`, `kotlin.math.tan`.
- **Beyond brief (cleanup):** rewrote the now-stale `newLatLngBounds`/`LatLngBounds` comments (the
  multi-pin block comment + the `mapSizePx` comment) to describe the v10 Mercator approach.

Debug APK built at `app/build/outputs/apk/debug/app-debug.apk` (~33 MB) for on-device A53 testing via
wireless debugging. **Blocker for on-device test:** debug build hits the real RevenueCat paywall (key
