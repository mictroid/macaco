## vc54 PUBLISHED to closed testing — Play photo/video permissions declaration unblocked

vc54 (the video feature go-live) was **dispatched 2026-07-03** but the WIF run
(`28685107862`) **failed** — not in the build. The AAB compiled, signed, and uploaded fine; the run
died at the very last step, `:commitEditForComDotHouseofmmminqDotMacaco`, with a **403
PERMISSION_DENIED** from the Play Developer API:

> "All developers requesting access to the photo and video permissions are required to tell Google
> Play about the core functionality of their app."

**Root cause:** vc54 is the first release requesting `READ_MEDIA_VIDEO` (added for the video
feature — it's correctly declared in `AndroidManifest.xml` alongside `READ_MEDIA_IMAGES`,
`RECORD_AUDIO`, etc.; no manifest change was needed). That permission makes Play require the
mandatory **Photo and Video Permissions declaration** (Play Console → App content). Until it's
submitted, `publishReleaseBundle` uploads the bundle but 403s at the final commit. The earlier
vc50–vc53 runs succeeded precisely because none of them added a media permission.

**Chicken-and-egg:** Play won't surface the *video* declaration question until it has *processed* a
bundle requesting `READ_MEDIA_VIDEO`, but the API commit fails before Play processes anything, so
re-dispatching CI just repeats the 403 forever. The user confirmed the video option wasn't yet
available to declare in the Console.

### How it was broken

1. **Temporarily swapped `release.yml`'s last step** from `./gradlew publishReleaseBundle` to
   `./gradlew bundleRelease` + an `actions/upload-artifact@v4` step (artifact name `app-release-aab`).
   `bundleRelease` still produces a **signed** AAB because the workflow already writes
   `keystore.properties` from the `RELEASE_KEYSTORE_*` secrets. Commit `c2fdf02`, pushed.
2. **Dispatched** run `28698669783` (07:08:37 UTC) → clean **green** run (no publish, no 403).
3. `gh run download 28698669783 -n app-release-aab` — pulled the signed AAB (~23.7 MB; CI does not
   archive it by default).
4. **Uploaded the AAB manually via the Play Console UI** → Closed testing → new release → Save as
   draft. Play processed the bundle, detected `READ_MEDIA_VIDEO`, and the declaration form then
   exposed the video question.
5. Completed the **Photo and Video Permissions declaration**, then rolled the release out from the
   UI. **vc54 is now PUBLISHED to closed testing.**
6. **Reverted `release.yml`** back to `publishReleaseBundle` (commit `eafe32c`, pushed; origin ==
   HEAD verified). Temporary `_aab_download/` folder deleted from the repo root.

### Why this is a one-time unblock

The declaration is **app-content-level and persists**, so future releases requesting the same
permission publish normally through the WIF CI path — no repeat of the manual dance is needed.
