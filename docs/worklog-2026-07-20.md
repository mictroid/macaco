### `feature/drive-dedupe-position-collision` ‚Äî commit `ad7f738` ‚Äî [PR #1](https://github.com/mictroid/macaco/pull/1)

Implements `docs/DONE/code-brief-drive-dedupe-position-collision.md`. Fixes the vc74 Drive-dedupe

### `feature/photo-fallback-premium-lapse` ‚Äî commit `6405d23` (+ `6d178c5` worklog) ‚Äî [PR #2](https://github.com/mictroid/macaco/pull/2)

Implements `docs/DONE/code-brief-photo-fallback-premium-lapse.md`. `JournalViewModel`'s auto
`downloadMissingPhotos` call was gated behind `isPurchased == true`, so a lapsed premium/trial
account stopped restoring Drive-backed photos whose local URI doesn't resolve on-device (cross-device
entries, or a removed gallery copy) ‚Äî even though those photos were legitimately uploaded while the
account was premium. Removed the gate on this read-only restore path (grants no new capability);
upload/connect paths (`saveEntry`, `syncPhotosToGoogleDrive`, `SettingsScreen`'s `DriveBackupCard`)
remain premium-gated, unchanged.

- **File:** `JournalViewModel.kt` only.
- **Deviations from brief:** none ‚Äî implemented exactly as specified.
- **Verification:** `./gradlew compileDebugKotlin` passes. On-device confirmation (lapsed account
  still sees previously-synced photos; upload/connect still gated) **not yet run** ‚Äî do before merge.
