## On-device signed-release QA pass — found and fixed a real R8 gap (`app/proguard-rules.pro`)

Built the signed release APK (`assembleRelease`) and installed on the S8+ (`ce03171353d9547e0c`) for
the `enable-r8` brief's required QA pass. **App crashed on launch, before `MainActivity` ever rendered**
— confirms the brief's own warning that "R8 bugs are runtime-only." `adb logcat` showed:

```
FATAL EXCEPTION: main
java.lang.RuntimeException: Unable to get provider androidx.startup.InitializationProvider
Caused by: java.lang.NoSuchMethodException: androidx.work.impl.WorkDatabase_Impl.<init> []
```

**Deviation from the brief:** the brief's keep rules covered the Google Drive API client and
kotlinx.serialization as reflection risks but missed **WorkManager** (used by `ReminderScheduler` for
periodic "write an entry" reminders), which persists scheduled work via an internal Room database.
Room instantiates its generated `*_Impl` class via `Class.getDeclaredConstructor()` reflection — R8 was
stripping `WorkDatabase_Impl`'s no-arg constructor. Added, not in the original brief:

```proguard
-keep class * extends androidx.room.RoomDatabase
-keep class androidx.work.impl.** { *; }
-dontwarn androidx.work.**
```

Rebuilt, reinstalled, relaunched — no crash, app opens cleanly. **Still need to complete the rest of the
brief's 6-point QA checklist** (Drive sync, local backup export/import, RevenueCat paywall/purchase/
restore, maps, Crashlytics deobfuscation) before shipping — only the launch-crash was caught and fixed
so far.
