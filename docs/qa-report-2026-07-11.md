## BLOCKER — ship-stopping

### B1. Video-only entries never download from Drive (multi-device / reinstall data loss) **[BRIEF: code-brief-drive-video-download-fix.md]**

`DrivePhotoSync.downloadMissing()` — the single code path that pulls Drive-backed media onto a
device that doesn't have it locally — returns early the instant there are no *photos* to fetch,
**before** it ever evaluates videos:

```kotlin
val needed = entries.flatMap { …photos… }.distinct()
if (needed.isEmpty()) return@withContext          // ← premature: kills video-only entries
val neededVideos = entries.flatMap { …videos… }.distinct()
if (needed.isEmpty() && neededVideos.isEmpty()) return@withContext   // the correct combined guard
```

Consequence: an entry that has videos but no photos (or whose photos are already cached) never
gets its videos downloaded. Repro: create a video-only entry on phone A → it syncs to phone B via
Firestore, but the video's local URI is dead on B and the Drive download is skipped → **the clip
shows nothing on B and after any reinstall.** The same `downloadMissing()` also feeds
`ensurePhotosCached()`, so a video-only entry is silently dropped from **backup export** too.

This directly undermines the headlined video feature and its "survives uninstall" promise. The
fix is a **one-line deletion** (remove the premature early return; the combined guard below
already handles the nothing-to-do case). Low risk, high impact.

## SHOULD-FIX — visible, not ship-stopping

### S1. Multi-plan paywall renders in English for all 10 non-English locales

The redesigned Monthly/Annual/Lifetime paywall added ~18 new `purchase_*` string keys to the
base catalogue that were **never translated**. `PurchaseScreen.kt` references all of them, so on
a German, French, Japanese, etc. device the buy screen mixes translated chrome with English plan
copy:

| Key | English value (shown untranslated everywhere) |
|-----|-----|
| `purchase_plan_monthly` / `_annual` / `_lifetime` | Monthly / Annual / Lifetime |
| `purchase_per_month` / `_per_year` / `_once` | / month · / year · one-time |
| `purchase_own_forever` | Pay once, own forever |
| `purchase_best_value` / `_most_popular` | Best Value / Most popular |
| `purchase_save_50` | Save 50% vs monthly |
| `purchase_free_trial` / `_trial_note` | 7 days free, then %s / year · Cancel anytime during trial… |
| `purchase_cancel_anytime` | Cancel anytime |
| `purchase_cta_monthly` / `_annual` / `_lifetime` | Start Monthly — %s · Try Free for 7 Days · Get Lifetime — %s |

English fallback means nothing *breaks*, but the most conversion-sensitive screen looks
half-finished in every translated market. Recommend a focused translation pass before a wide
international launch. (I can generate the full 10-language translation brief on request.)

### S2. Broader locale drift — Swedish is 47 strings behind

Every translated locale is missing ≥18 base strings; **Swedish (`values-sv`) is missing 47**.
Beyond the paywall (S1), assorted newer strings fall back to English in translated builds. Worth
a catalogue-completeness pass; Swedish should be prioritised or the language temporarily delisted
if it can't be completed.

| Locale | Missing keys |
|--------|--------------|
| de | 18 |
| es, fr, it, ja, nl, pl, pt, zh-rCN | 20 each |
| **sv** | **47** |
