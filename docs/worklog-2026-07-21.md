## Header brand-icon centering fix (`code-brief-header-icon-centering.md`)

### Problem
The branded header `macaco` icon didn't hold the splash screen's centered position — it shifted
between the expanded and scroll-collapsed header states, and sat differently on some screens
(Adventures/Map worst) than others (Profile, the "good" reference). Root cause: each screen
hand-rolls its own header `Box` around `MacacoBrandBlock`, and they disagreed on whether that outer
container carried `windowInsetsPadding(WindowInsets.safeDrawing.only(Horizontal))`. Since
`MacacoBrandBlock` centers within whatever width its parent gives it, that one line changed which
width the icon centered against. Map applied it *conditionally* (only when `!hasMovedMap`), so the
icon jumped sideways the instant the header collapsed on pan/zoom.

### Fix (per brief — icon always centers on TRUE screen width, matching splash/Profile)
- **MapScreen.kt** — removed the collapse-conditional `.then(if (hasMovedMap) … else Horizontal
  inset)` on the outer header `Box`; moved nav-bar clearance onto the landscape trailing label by
  wrapping its `Text`s in a `Row` with the `End` inset. Icon no longer jumps on collapse.
- **JournalListScreen.kt** — removed the `Horizontal` inset from the header `Box`; added `Start`/`End`
  insets to the landscape leading search-button `Box` and trailing avatar `Box`.
- **HelpAboutScreen.kt** — removed the `Horizontal` inset; added the `Start` inset to both the
  landscape and portrait back-button `IconButton`s.
- **EntryDetailScreen.kt** — removed the `Horizontal` inset; added `Start`/`End` insets to the
  leading (back+counter) and trailing (share/edit/delete) `Row`s.
