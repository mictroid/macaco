## landscape-layout-consistency brief (7 changes, 4 files + 1 new) — IMPLEMENTED. `assembleDebug` SUCCESSFUL. Brief → `docs/DONE/`. Not yet shipped.

Single large Cowork brief making the journal list, Help & About, Profile, and Entry Detail behave
consistently in landscape and on tablets. All target files verified vs live source first; every BEFORE
snippet matched. `assembleDebug` green (1m03s).

- **Change 1 — `ui/theme/ContentWidth.kt` (new)** — shared 840 dp max-content-width token
  (`MacacoContentMaxWidth`) + `macacoContentGutter(min=16.dp)` that centres content once the screen is
  wider than the cap. File already existed untracked from an earlier session; content matched the brief
  verbatim, so it was left as-is and picked up by this commit.
- **Change 2 — `JournalListScreen.kt`** — list gutter replaced the ad-hoc `if (screenWidthDp>=600) 80.dp
  else 16.dp` with `macacoContentGutter()`. Added `import …ui.theme.macacoContentGutter`.
- **Change 5 — `JournalListScreen.kt`** — landscape header now scroll-hides: outer branded Box keeps the
  status-bar brand strip + gains `windowInsetsPadding(safeDrawing Horizontal)`; the existing compact/tall
  `if/else` moved inside a new `AnimatedVisibility(visible = !(isLandscape && collapsed))` → inner
  animateContentSize Box (two extra closing braces). Added imports `AnimatedVisibility`,
  `expandVertically`, `shrinkVertically`, `safeDrawing`.
- **Change 3 — `HelpAboutScreen.kt`** — content column scroll state hoisted (`scrollState`) and gutter
  switched to `padding(horizontal = macacoContentGutter(), vertical = 16.dp)`.
- **Change 6 — `HelpAboutScreen.kt`** — `TopAppBar` + in-content brand band replaced by a journal-style
  collapsing branded header (tall brand block at rest, slim centred row when scrolled/landscape,
  scroll-hides in landscape). Added the animation/inset/`derivedStateOf`/`LocalConfiguration` imports;
  **removed now-unused `TopAppBar`, `TopAppBarDefaults`, and `draw.clip` imports** (clip was only on the
  deleted band).
- **Change 4 — `ProfileScreen.kt`** — horizontal `safeDrawing` insets re-applied to the four content
  siblings below the edge-to-edge banner (identity column, divider, action-tile grid, Delete Account
  button) to fix the A53 3-button side-nav overlap that the single-column rework had dropped. Added
  `WindowInsetsSides`/`only`/`safeDrawing`/`windowInsetsPadding` imports (`WindowInsets` already present).
  **Deviation per the brief's own note:** also applied the same inset to the "Member since" footer, the
  one further sibling below Delete Account.
- **Change 7 — `EntryDetailScreen.kt`** — full-height themed `TopAppBar` replaced by a slim ~48 dp

  weighted spacers, wordmark drops below 420 dp wide, share/edit/delete trail). A brand moment under the
  chrome policy, so the fixed splash palette is intended. Added `Image`, `painterResource`, `offset`, and
  the `WindowInsets` companion imports; **removed now-unused `TopAppBar`/`TopAppBarDefaults` imports**.
